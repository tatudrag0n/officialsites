Minecraft Server

みんなでつくる、統合型Minecraftサーバー。

MinerVaは、EM経済、クエスト、ショップ、オークション、ミニゲーム、保護システムを一体化したMinecraftサーバーです。サバイバルだけでなく、取引・挑戦・交流まで含めて長期的に遊べる構成を目指します。

経済と遊びをつなぐMinecraftサーバー。
Features

MinerVaの主な要素

遊び方を単発で終わらせず、サーバー内の行動が経済・称号・クエストに接続される設計です。

Economy

EM経済

1エメラルド=100EMを基準に、Mob討伐、クエスト、取引、イベントなどからEMを獲得。価格は入手難度・希少性・経済影響を考慮して調整します。

Quest

デイリー / ウィークリー / マンスリー / スペシャル

ステータス本から進捗を確認し、条件達成でEMを受け取るクエストシステムを搭載予定。日課・週課・長期目標を分けて設計します。

Shop

ショップとオークション

メインショップ、棚ショップ、樽ショップ、商人、額縁オークションを用意。低〜中価値品の流通と希少品の競売を分けます。

Status

ステータス / 称号 / 転生

プレイ時間、討伐、建築、経済、進捗などを可視化。やり込みに応じて称号や転生などの長期目標へ接続します。

Protection

保護チャンク

専用ビーコンや保護機能で、建築物・コンテナ・ショップなどを保護。公共施設や中央エリアの安全性も確保します。

Games

アスレ / ミニゲーム / FFA

サバイバル以外の遊び場として、アスレチック、ミニゲーム、FFAを設置。報酬とクエストで参加理由を作ります。

EM System

経済は「稼ぎやすさ」より「崩れにくさ」を重視。

Mob報酬はゾンビ=10EMを基準にしつつ、自動化・トラップ・過剰供給で経済が壊れないよう制御します。進捗報酬は一回限りとし、恒久倍率は廃止する方針です。

100EMエメラルド1個を基準
10EMゾンビ討伐の基準報酬
4種デイリー / ウィークリー / マンスリー / スペシャル
5〜10%オークション手数料想定
Shop Design

ショップ構成

万能ショップ化を避け、役割ごとに流通経路を分けます。

メインショップ基本アイテムを安定供給する基準ショップ。価格表のPrice_EMを基準にします。
棚ショップ棚を使った独自ショップ。1つの棚で複数アイテムを販売できる導線を想定。
樽ショップジャンク品・余剰品・低〜中価値素材をランダム販売。高級資源や特殊品は原則除外。
商人赤・青・黄・紫などの種類を分け、販売・買取対象を制御。高難度・希少品ほど低確率にします。
額縁オークションプレイヤー出品物、イベント品、希少品などを入札形式で流通。手数料をEMシンクとして利用。

Step 03
